# Google Photos Takeout Plan

Google Photos and Google Takeout exports should be treated as import sources that need normalization before enrichment.

## Expected Inputs

- Takeout folder exports.
- Media files such as JPG, HEIC, PNG, MP4, MOV.
- JSON sidecar files generated by Google Takeout.
- Album folders and duplicated media references.

## Planned Flow

1. Scan Takeout folder read-only.
2. Detect media files and JSON sidecars.
3. Link sidecars to likely media files.
4. Preserve original Takeout paths.
5. Write normalized inventory records.
6. Queue review for duplicates, missing metadata, and privacy classification.

## Current Parser

```bash
python3 scripts/parse_google_takeout.py --path "/path/to/Takeout"
```

This creates:

- `data/google_takeout_inventory.csv`
- `data/google_takeout_inventory.json`
- `data/google_takeout_sidecars.json`

## Sidecar Handling

Future import logic should parse Google JSON metadata using structured JSON parsing and should never overwrite embedded media metadata without review.

## Risks

- Same media may appear in multiple album folders.
- File timestamps may reflect export time instead of capture time.
- JSON sidecars may contain useful title, description, creation time, and geo fields.
- Privacy review is required before any publishing workflow.
